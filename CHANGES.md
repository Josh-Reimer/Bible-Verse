# Recent Changes

## Edge-to-edge layout fixes (`b4a9d83`)

**Problem:** `VerseLookUpActivity` had two layout issues:
- A white/black strip appeared above the navigation gesture bar because `ScrollView` was clipping its bottom padding area with the default `clipToPadding="true"`.
- The back arrow in the toolbar was clipped because the toolbar had a fixed `?attr/actionBarSize` height that didn't grow to absorb the status-bar top inset applied in code.

**Fix:**
- Added `android:clipToPadding="false"` to the `ScrollView` in `verse_lookup_activity.xml` ā€” content now draws behind the nav bar while scrolling, removing the visible strip.
- Changed toolbar `android:layout_height` from `?attr/actionBarSize` to `wrap_content` with `android:minHeight="?attr/actionBarSize"` ā€” the toolbar now expands to fit both its normal height and the status-bar padding, keeping the back arrow fully visible.

## Multi-translation support ā€” KJV, ASV, BSB (`fa1bf57`)

**What changed:**

- All 66 KJV book files moved from `assets/` root into `assets/kjv/`.
- 66 ASV (American Standard Version, 1901, public domain) book files added to `assets/asv/`, sourced from `scrollmapper/bible_databases`.
- 66 BSB (Berean Standard Bible, 2023, public domain) book files added to `assets/bsb/`, sourced from `scrollmapper/bible_databases`.
- `Tools.getFile()` updated to read the `"translation"` key from `SharedPreferences("settings")` (default `"kjv"`) and prepend it as a folder prefix. This single change propagates to every asset read in the app.
- `Tools.getFile()` also switched from the old `available()` + single `read()` pattern to `InputStream.readAllBytes()`, which reliably reads compressed assets in subdirectories.
- `Bible.getBookLength()` hardened with `stripTrailing()` before parsing the last line ā€” KJV files have no trailing newline but ASV/BSB files do, which previously caused a `NumberFormatException`.
- `SettingsActivity` gained a Translation `Spinner` (KJV / ASV / BSB) below the dark-theme switch. Selection is saved to `SharedPreferences("settings")` under key `"translation"`.
- `MainActivity.onResume()` now re-fetches and re-displays the current verse, so a translation change in Settings applies immediately on back-navigate without requiring a full restart.

## Red-letter words of Christ for KJV (`02a8c38`)

**What changed:**

- `assets/red_letter_kjv.json` ā€” 2027 KJV verses containing words spoken by Jesus, stored as HTML strings with `<font color="#CC0000">` wrapping the red-letter portions. Generated by parsing the CrossWire KJV OSIS XML (Klopsch 1901 edition) which uses `<q who="Jesus">` elements at word/phrase level. Cross-verse quote spans (where a `<q>` element spans multiple verse milestones) are handled correctly.
- `RedLetter.java` ā€” loads the JSON lazily on first use and exposes `getSpanned(context, verseRef)`. Returns a `Spanned` (via `Html.fromHtml()`) for KJV red-letter verses, `null` for all other translations or verses with no red-letter content.
- `MainActivity` ā€” added `showVerse(Verse)` helper that calls `RedLetter.getSpanned()` and uses `TextView.append(Spanned)` for red-letter verses, or falls back to `setText(full_text)` for plain text.
- `VerseLookUpActivity` ā€” switched pre-verse and post-verse accumulation from plain `String` concatenation to `SpannableStringBuilder`, so red-letter markup is applied per-verse across all three text sections (before, target, after). The target verse uses the existing green/bold style with red-letter spans overlaid.
